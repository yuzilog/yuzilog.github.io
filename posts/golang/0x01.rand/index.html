0x01.随机数的库

这篇文章主要介绍使用golang的随机函数库包括以下两种

  • math/rand 伪随机
  • crypto/rand

math/rand

Package rand implements pseudo-random number generators unsuitable for security-sensitive work. 包的说明中，明确指出使用的伪随机数，seed固定时，生成的序列值是固定的。也就是说，只要传入的seed固定，每次程序重新启动函数的序列就是一样的。

func TestRand(t *testing.T) {
	rand.Seed(1)
	for i := 0; i < 4; i++ {
		println(rand.Intn(100))
	}
}

输出列表
=== RUN   TestRand
81
87
47
59
--- PASS: TestRand (0.00s)

golang中默认seed值为1，依据seed的值值输出不同随机数序列，程序每次重启后，序列值则相同，所以使用的时候需要注意。 简单是的使用场景，则是在每次使用随机数时，使用时间戳生成随机数，如下：

rand.Seed(time.Now().UnixNano()) println(rand.Intn(100))

crypto/rand

crypto/rand是为了提供更好的随机性满足密码对随机数的要求，以Linux为例，读取操作系统的中/dev/urandom /dev/random和/dev/urandom是Linux系统中提供的随机伪设备，这两个设备的任务，是提供永不为空的随机字节数据流。很多解密程序与安全应用程序（如SSH Keys,SSL Keys等）需要它们提供的随机数据流。 /dev/random:依赖于系统中断，在系统的中断数不足时，/dev/random设备会一直封锁，尝试读取的进程就会进入等待状态，直到系统的中断数充分够用, /dev/random设备可以保证数据的随机性。 /dev/urandom:不依赖系统的中断，不会造成进程忙等待，但是数据的随机性也不高。

func TestRand(t *testing.T) {
	// rand.Seed(1)
	// for i := 0; i < 4; i++ {
	// 	println(rand.Intn(100))
	// }
	for i := 0; i < 4; i++ {
		n, _ := rand.Int(rand.Reader, big.NewInt(100))
		println(n.Int64())
	}
}

性能差别 相同的运行环境，差别超过10倍

func BenchmarkCryptoRand(b *testing.B) {
	for i := 0; i < b.N; i++ {
		crand.Int(crand.Reader, big.NewInt(100))
	}
}

func BenchmarkMathRand(b *testing.B) {
	rand.Seed(10)
	for i := 0; i < b.N; i++ {
		rand.Intn(100)
	}
}

测试结果如下：
BenchmarkMathRand-4   	49438474	        24.51 ns/op	       0 B/op	       0 allocs/op
BenchmarkCryptoRand-4   	 861868	      1182 ns/op	      56 B/op	       4 allocs/op

总结

  • 对于不涉及到密码类的开发工作直接使用math/rand+基于时间戳的种子rand.Seed(time.Now().UnixNano())一般都能满足需求
  • 对于涉及密码类的开发工作一定要用crypto/rand
